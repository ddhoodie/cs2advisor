TEMPLATES

TMPL-1
Grupa B — Round type decision (6 pravila → 1 template, 6 instanci)
R-B01, R-B02, R-B03, R-B04, R-B05, R-B06 su identični po obliku — razlikuju se samo u side, threshold, count_op i round_type.
Template ROUND_TYPE_DECISION(side, threshold, count_op, n, result_type)
when round.type == UNDEFINED
AND player.side == {side}
AND team.players_above_threshold({threshold}) {count_op} {n}
then round.type ← {result_type}
TMPL-2
Grupa C — ECO budget (R-C01 + R-C02 → 1 template, 2 instance)
Isti obrazac: ECO runda + strana + prag. Razlikuju se samo side i threshold.
Template ECO_BUDGET(side, threshold)
when round.type == ECO AND player.side == {side}
then spend_budget ← player.money − max(0, {threshold} − player.loss_bonus)
TMPL-3
Grupa G — AWP availability block (R-G01 + R-G02 su sličnog oblika)
Oba provjeravaju stranu i broj AWP-ova u timu, samo se prag broja razlikuje.
Template AWP_BLOCK(side, awp_count_op, n)
when player.side == {side}
AND team.awp_count {awp_count_op} {n}
then AWP.available ← false

Pravila grupe A imaju viši prioritet od grupe B. Ako je bilo koje pravilo iz grupe A okidača, pravila grupe B se ne evaluiraju. Specijalne runde uvek rezultuju BUY rundom bez obzira na stanje tima.

GRUPA J Fallback — najskuplje dostupno oružje
Prioritet: Nizak (fallback)
Pravilo J01 aktivira se jedino kada nijedno pravilo iz grupa G i I nije postavilo purchase.weapon. Iz skupa oružja koji je filtriran grupama F i H, bira se najskuplje dostupno oružje — bez obzira na stranu, playstyle ili pozicioniranje. Deagle je u skupu kandidata samo ako R-H02 nije blokiran.
R-J01Fallback — kupi najskuplje dostupno oružje
Fallback
Pre-uslov: purchase.weapon == UNDEFINED (nijedna prethodna grupa nije postavila oružje)
When (Uslov) IF purchase.weapon == UNDEFINED AND ∃ w : w.available == true
Then (Akcija) THEN purchase.weapon ← most_expensive( {w | w.available == true} )

Primer — BC-5: EXPLAIN(purchase.weapon = AK47)
Igrač T strane, BUY runda, weapon_budget = 2750. Sistem je odabrao AK47 putem R-I03. BC rekonstruiše ceo lanac zavisnosti od ulaznih podataka.
BC-5 / Nivo 1 Zašto purchase.weapon = AK47?
BUYT strana
Izvedeno pravilom R-I03 // T — AK47 (2700–2999, nije utility) Uslovi koje treba objasniti: round.type == BUY ← izvedena player.side == T ← inicijalna ✓ weapon_budget ∈ [2700,3000) ← izvedena is_utility_player == false ← inicijalna ✓
Šta se objašnjava dalje → EXPLAIN(round.type = BUY) → EXPLAIN(weapon_budget = 2750) // player.side i is_utility_player // su inicijalne — nema rekurzije
BC-5 / Nivo 2a Zašto round.type = BUY?
Rekurzija
Izvedeno pravilom R-B01 // T — BUY (≥ 4 igrača iznad 3700) Uslovi: round.type == UNDEFINED ← pre-uslov ✓ player.side == T ← inicijalna ✓ players_above(3700) ≥ 4 ← inicijalna ✓ // Nijedna A-grupa pravila // nije aktivirana pre B-grupe
Svi uslovi su inicijalni STOP — rekurzija završena // player.side = T (ulaz) // players_above(3700) = 4 (ulaz) // round.type bio UNDEFINED (pre-uslov)
BC-5 / Nivo 2b Zašto weapon_budget = 2750?
Rekurzija
Izvedeno pravilom R-D01 // T — uvek pun oklop Akcija koja je postavila vrednost: purchase.armor ← FULL weapon_budget ← player.money − 1000 = 3750 − 1000 = 2750
Uslovi pravila R-D01 round.type == BUY ← izvedena (već objašnjena) player.side == T ← inicijalna ✓ player.money = 3750 ← inicijalna ✓ // STOP — sve inicijalne ili već // objašnjene u nivo 2a
